# ファイル操作ポリシー

作成日: 2026-06-22

## 1. 基本ルール

- 既存ファイルの変更前に、対象ファイルの役割を確認する。
- 既存仕様書本文は、根拠と対象範囲を明示してから変更する。
- 自動生成ファイルと手書き仕様書を混在させる場合は、生成元を明記する。

## 2. 追加

以下は追加してよい。

- 新しい仕様変更案
- レポート
- 会話ログ
- アプリ部署ルール
- AI 社員ルール
- テンプレート

## 3. 修正

修正時は、変更理由を本文または変更案に残す。

仕様書を修正する場合は、次を確認する。

- 既存仕様書の記述
- 関連する実装コード
- 関連する運用メモ
- 直近の会話ログ

## 4. 削除

AI 社員は直接削除しない。

削除相当の変更は、次の形式で提案する。

```text
status: deprecated
deprecated_reason: ...
replaced_by: ...
source_conversation_id: ...
```
