## ユーザー指示

## 反映内容

### 初回表示と初期設定ウィザード

- 初回起動時は初期設定ウィザードを直接表示しない。
- まず「親方メモへようこそ」画面を表示する。
- 「初期設定をする」を選んだ場合のみ初期設定ウィザードを開く。
- 「あとで」を選んだ場合は作業タブへ進む。
- 請求書PDF作成など初期設定が必要な操作では、初期設定ウィザードまたは請求書設定へ誘導する。

### 表記統一

- ユーザー向けボタン名を「サンプルで試す」に統一する。
- 内部実装名は tutorial / sample flow を使ってよい。
- Google Play上の表示名は「現場メモ日報」、アプリ内説明では愛称として「親方メモ」を使ってよい。

### いつもの作業

- 初期実装では「いつもの作業」より先に「前回と同じで即保存」を実装する。
- 「いつもの作業」を保存する場合は、将来 `usual_works` 相当のDBテーブル追加を検討する。

### 請求取消・再請求

- 請求取消は請求書単位で行う。
- 請求履歴は物理削除せず、状態を変更して残す。
- 請求取消を行うと、その請求書に含まれる作業記録はすべて未請求へ戻る。
- 作業単位で請求済みを未請求へ戻す操作は原則行わない。
- 請求済み作業は削除できない。

### 非課税表示

- 既存作業登録画面の完了済みタスクと、新しい確認型作業記録画面・前回と同じで即保存の未実装タスクを分離した。

### Googleカレンダー連携

- 初期公開版ではGoogleカレンダー連携を行わない。
- v1.0以降の検討項目とし、実装時はOAuth、権限、プライバシーポリシー、データセーフティ申告を更新する。

### ROADMAP

- TASKSで完了済みのv0.4請求運用項目に合わせてチェック状態を更新した。
