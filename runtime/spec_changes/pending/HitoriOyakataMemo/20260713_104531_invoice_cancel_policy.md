# 仕様変更案: 請求取消と再請求の運用方針

追加日時: 2026-07-13T10:45:31+09:00  
更新日時: 2026-07-13T10:53:51+09:00  
対象アプリ: HitoriOyakataMemo  
区分: change / question  

## ユーザー指示

請求履歴・再請求の整合性改善について、次の方針を検討する。

- 請求済みの作業は削除できないようにする。
- 請求済み作業を未請求へ戻す場合、履歴整合性を優先し、同じ請求書に含まれる作業はすべて未請求へ戻す。
- 請求履歴には、未請求に戻した印を残す。
- 「請求解除」「請求破棄」などの用語候補があるが、より自然な言葉を検討する。
- 入力完了度は邪魔にならないようにする。

## 用語方針案

操作名は「請求を取り消す」、状態表示は「請求取消済み」を基本候補にする。

理由:

- 「請求破棄」は履歴や証跡も捨てる印象が強く、履歴を残す方針と相性が悪い。
- 「請求解除」はシステム操作としては理解できるが、利用者向けにはやや硬い。
- 「請求取消」は、発行済みの請求を無効化したことが比較的伝わりやすい。

## 追加したタスク

- 請求済み作業を削除できないようにする。
- 削除しようとした場合は、先に請求を取り消す必要があることを表示する。
- 請求履歴に「発行済み」「請求取消済み」「再請求済み」などの状態を表示する。
- 請求取消は請求書（請求履歴）単位で行い、同じ請求書に含まれる作業はすべて未請求へ戻す。
- 1件だけ戻したい場合でも、履歴整合性を優先して同じ請求書内の全作業が未請求へ戻ることを説明する。
- 請求取消の確認ダイアログに、対象件数、対象金額、請求番号を表示する。
- 請求取消後の再請求では、必要な明細だけを選び直して請求書を作成できるようにする。
- 入力完了度は常時大きく表示せず、入力状況チップや小さなチェック表示を優先して検討する。

## 承認が必要な理由

この仕様は、請求済みステータス、削除可否、請求履歴、DBスキーマに影響する可能性がある。
実装前に、履歴状態の保存方式と請求書単位の請求取消ルールを確定する必要がある。
