## ユーザー指示

再請求、作業登録など、利用者がやりたいアプリ操作のパターンを体系化し、後で操作マニュアルを作るための別ファイルとしてフローを作成する。

## 作成ファイル

- `AndoroAPP/HitoriOyakataMemo/docs/USER_OPERATION_FLOWS.md`

## 整理した主な操作フロー

- 毎日の作業を記録する
- 前回と同じ内容で記録する
- いつもの作業から記録する
- 取引先を登録する
- 現場を登録・完了する
- 残業時間を入力する
- 一式工事・材料費・諸経費・値引きを記録する
- 作業記録を編集・削除する
- 月末に未請求を確認する
- 請求書を作成する
- 一部の明細だけ請求する
- 誤って請求した作業を未請求に戻す
- 再請求する
- 請求履歴を確認する
- PDF請求書を保存する
- CSVを出力する
- バックアップ・復元する
- プレミアムプランを確認する
- 初回起動時に流れを試す
- Googleカレンダー予定から作業記録を作る

## 方針

- 画面ごとの説明ではなく、ユーザーがやりたいこと起点で整理する。
- 実装済み、実装予定、検討中を区別する。
- 後で `docs/USER_MANUAL.md` を更新する際の元資料として使う。
