## 追加したタスク

### 残業時間入力

- 内部保存は既存仕様通り分単位のまま維持する。
- 入力UIでは「時間」と「分」の両方で入力できるようにする。
- 例: `1時間30分` と `90分` のどちらでも扱えるようにする。
- CSV、PDF、計算処理は分単位の値を使い続ける。

### 入力完了度

- 初回ユーザー向けに入力完了度の表示を検討する。
- 必須項目と任意項目を分けて算出する。
- 日々の記録では邪魔にならないよう、詳細または補助表示にする。
- パーセンテージ表示、チェックリスト表示、入力状況チップを比較する。
- 「完璧に埋めないと保存できない」と誤解されない文言にする。

### 請求履歴・再請求

- 請求済み作業を未請求へ戻して再請求できる現在方針を画面上で説明する。
- 請求履歴に「発行済み」「取消済み」「再請求済み」などの状態表示を検討する。
- 発行済み取消時は履歴を物理削除せず、取消済みとして残す方式を検討する。
- 再請求時に元の請求履歴と新しい請求履歴を関連付けるか検討する。
- カレンダーや請求履歴で、有効な請求情報と不要になった請求情報を判別できる表示を検討する。

### 説明・マニュアル

- 残業時間入力、詳細項目、誤請求時の再請求、請求履歴を残す方針を説明へ反映する。
