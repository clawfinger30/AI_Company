# XserverVPS 承認が必要な操作一覧

作成日: 2026-07-02

## 1. 基本方針

XserverVPSは本番環境を含むため、外部公開、認証、DB、証明書、Firewall、サービス稼働に影響する操作は承認対象とする。

承認は、対象、目的、実行予定コマンド、影響範囲、検証方法、ロールバック方法が揃ってから求める。

## 2. 承認不要

- ファイル一覧、Git状態、ログ、リソース状況の読み取り調査
- 仕様書、README、設定ファイルの構造確認
- 変更案、手順案、ロールバック案の作成
- 実行しないコマンド案の提示

## 3. 通常承認が必要

- リモート本番ファイルの追加、変更、移動
- Git操作による作業ツリー変更
- Docker imageのpull、build、再起動、停止
- Nginx設定反映、reload、restart
- systemdサービスのrestart、reload、enable、disable
- cron、system timer、定期処理の変更
- パッケージインストール、アップデート
- 証明書更新、SSL設定変更
- Firewall、ポート、公開URL、リバースプロキシの変更
- `.env`、環境変数、シークレット参照方法の変更
- DBマイグレーション、バックアップ、リストア

## 4. 強い承認が必要

以下は、通常承認よりも慎重に扱い、可能な限りバックアップ、停止時間、復旧手順を先に確定する。

- DB削除、テーブル削除、大量更新
- Docker volume削除
- SSH設定変更
- ユーザー、sudoers、権限、所有者の変更
- FirewallでSSH接続に影響する変更
- 証明書秘密鍵、APIキー、パスワードの更新
- `git reset --hard`、`git clean -fdx`
- サービス全停止を伴う作業

## 5. 承認記録に残す項目

- 承認者
- 承認日時
- 対象サーバー
- 対象パス
- 対象サービス
- 実行予定コマンド
- 想定影響
- 検証方法
- ロールバック方法
- 実行結果
