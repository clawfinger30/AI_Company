# 仕様書メモAI ワークフロー

作成日: 2026-06-22

## 1. 自動記録

仕様書メモAIは、ユーザーから明示的な「仕様メモ」依頼がなくても、仕様影響がある会話、調査、コード変更を検知した場合に自動で記録する。

自動記録の対象:

- ユーザーが新しい仕様、仕様変更、仕様廃止、運用判断を指示した場合
- 実装変更により画面、API、DB、設定、外部連携、スケジュール、認証、通知、投稿処理などの挙動が変わる場合
- 調査中に、コードと既存仕様書の差分、未記載仕様、未確定事項を見つけた場合
- リリース、ビルド、バージョン更新、Gitコミットに関係する作業で、仕様書や履歴へ残すべき情報がある場合

自動記録しない対象:

- 単なる質問回答で、仕様、コード、運用履歴への影響がない場合
- ファイル閲覧だけで、新しい判断や差分がない場合
- 一時的な調査ログで、仕様として残す根拠がない場合

## 2. 手動起動

ユーザーが以下のように依頼した場合も起動する。

```text
仕様メモして
仕様書更新案を作って
この会話を仕様書に反映して
コードと仕様書が合っているか見て
```

## 3. 処理順

1. 全社共通ルールを読む
2. 仕様書メモAIの職種定義を読む
3. 対象アプリ部署ルールを読む
4. 対象アプリ専属の仕様書メモAIルールを読む
5. 会話ログを確認する
6. 既存仕様書を確認する
7. 関連コードを確認する
8. 仕様差分を分類する
9. バージョン、Git状態、対象ファイル、検証方法を確認する
10. 変更案またはレポートを作成する
11. 承認が必要な項目を分ける
12. `change-tracking-policy.md` の必須項目が埋まっているか確認する

## 4. 出力先

```text
00_ai_company/runtime/spec_changes/pending/{app}/{YYYYMMDD_HHmmss}_{summary}.md
00_ai_company/runtime/reports/{app}/{YYYYMMDD_HHmmss}_{summary}.md
```

会話ログを残す場合は、以下に追記する。

```text
00_ai_company/runtime/conversation_logs/{YYYY-MM-DD}.jsonl
```

## 5. 自動反映の扱い

自動記録は、会話ログ、仕様変更案、差分レポートの作成までとする。

仕様書本文への反映は、ユーザーが明示的に依頼した場合のみ行う。

## 6. 削除候補の扱い

削除候補は直接削除せず、次のいずれかで提案する。

- `deprecated`
- `archive_candidate`
- `replaced_by`

## 7. 追跡情報

仕様変更案またはレポートには、以下を必ず確認して記録する。

- 追加日時、更新日時
- 変更前後のアプリバージョン
- Git リポジトリ、ブランチ、コミットID
- 変更ファイル一覧
- コード確認結果
- 検証結果
- Git 管理外または未検証の場合の理由
